# Checklists

These are the quick "pause before you do the thing" checks.

Use them when a task is about to touch files, leave your machine, ask for credits, publish a repo, or use work that could be visible outside your team.

## Before Editing

Use `checklists/before-editing.md` before changing files.

It is the basic root, path, scope, and verification check.

## Before Submitting

Use `checklists/before-submitting.md` before sending forms, applications, grant notes, program responses, or anything similar.

It is the "what fields are leaving, and did we strip private data?" check.

## Before Publishing

Use `checklists/before-publishing.md` before making a repo, doc, release, post, or example public.

It is the public-copy check: synthetic examples, scanner review, limits, and no fake adoption claims.

## Before Asking For Review

Use `checklists/before-asking-for-review.md` before sending a repo link, review packet, or feedback ask to someone else.

It is the "one relevant person, one useful note, no private context, no spam" check.

## Before Claiming Traction

Use `checklists/before-claiming-traction.md` before saying people are reviewing, adopting, sharing, or using a repo.

It is the "say what is visible, do not turn quiet into momentum" check.

## Before Using Shared Tokens

Use `checklists/before-using-shared-tokens.md` before routing prompts through a data-sharing or eval/training opt-in project.

It is only for public or synthetic work. If the prompt would feel bad on someone else's desk, stop.

## Suggested Order

```text
editing -> submitting -> publishing -> asking for review -> claiming traction
```

Use the shared-token checklist only when that specific setting is part of the work.
