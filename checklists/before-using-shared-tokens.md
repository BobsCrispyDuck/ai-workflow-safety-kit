# Checklist: Before Using Shared Tokens

- Confirm data sharing is approved for the selected project only.
- Confirm the work is public or synthetic.
- Confirm the prompt contains no private project data.
- Confirm output may be used for training/evaluation.
- Do not use production logs.
- Do not use user data.
- Do not use secrets, keys, auth, billing, wallet, or session data.
- Do not make the shared-token key the global default.
- Stop if the prompt would be uncomfortable to share externally.
