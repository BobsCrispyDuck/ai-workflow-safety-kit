# Template: Public Release Check

Use before publishing a repo update, docs change, example, template, screenshot, demo, or announcement.

This is for the last pause before something leaves your machine.

## Release Target

```text
Thing being released:
Where it will go:
Who can see it:
Why it is useful:
```

## Content Check

Confirm the release does not include:

- secrets, API keys, or tokens
- private repo contents
- production logs or config
- customer or user data
- account, billing, wallet, session, or auth details
- raw assistant transcripts
- private creative work or client material
- machine paths, local usernames, or internal project names

## Claims Check

Confirm the release does not claim:

- work is tested when it is not
- production readiness without proof
- official endorsement without proof
- safety guarantees it cannot make
- results from private or sensitive data

## Proof Check

```text
Files reviewed:
Checks run:
What passed:
What still needs review:
```

## Approval Gate

If the release is external, irreversible, tied to an account, or likely to attract attention, stop for explicit approval.

```text
Approval needed: yes/no
Approval phrase:
Approved by:
Date:
```

## Release Receipt

```text
Released:
Destination:
Commit or version:
Checks run:
Known limits:
Next step:
```
