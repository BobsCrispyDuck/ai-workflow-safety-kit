# Support

This repo is a small public workflow-safety kit, not a support desk.

The most useful support request is a public-safe note that helps make the kit clearer.

## Best Public Paths

Use these when your note can be shared without private details:

- [reviewer scorecard](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=reviewer-scorecard.yml) - you tried one prompt or scenario
- [scenario idea](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=scenario-idea.yml) - the kit is missing a fake workflow case
- [safety miss](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=safety-miss.yml) - the assistant should have stopped but did not
- [docs or template feedback](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=docs-template-feedback.yml) - one line or page is confusing

If you are not sure, use `docs/issue-guide.md`.

## Do Not Post

Do not post:

- secrets or keys
- production logs or configs
- private repo names
- user or customer data
- account, billing, wallet, session, or auth details
- screenshots with private context
- raw assistant transcripts

If a real story matters, rewrite it with fake names and fake details first.

## Sensitive Reports

If your issue needs private material to explain it, do not open a public issue.

Read `SECURITY.md` first and keep the public note minimal and sanitized.

## What This Project Can Help With

Good fit:

- clearer templates
- stronger stop conditions
- missing synthetic scenarios
- better review prompts
- more useful receipts
- docs that are easier to follow

Bad fit:

- emergency production support
- account, billing, or API-key help
- legal, compliance, or security review
- debugging private repos
- reviewing real logs or transcripts
