# Sharing Guide

If this repo helped you slow down in the right place, share it like a useful tool, not a launch.

One small post in the right place is better than spraying the same link everywhere.

## Short Version

```text
Small repo for coding-agent guardrails:

https://github.com/BobsCrispyDuck/ai-workflow-safety-kit

It is templates, checklists, fake scenarios, and receipts for the boring failure modes: wrong folder, private context in public work, external action without approval, and "done" without proof.
```

## Where It Fits

Good places:

- a thread about coding-agent workflow
- a repo issue asking for safer AI-assisted contribution patterns
- a small team chat where people already use coding assistants
- a public discussion about eval scenarios or guardrails

Bad places:

- unrelated promo threads
- every channel at once
- replies to people asking for product support
- places where a link-only drop is considered spam

## What To Ask For

Ask for one useful note, not applause.

```text
If you try it, I would rather have one missing failure mode than a general thumbs-up.
```

Useful asks:

- read `docs/faq.md`
- read `docs/reviewer-brief.md`
- try one prompt from `docs/reviewer-prompts.md`
- fill out `docs/reviewer-scorecard.md`
- suggest one fake scenario for `evals/scenarios.jsonl`
- point out one checklist line that is vague

Direct public-safe feedback links:

- [reviewer scorecard](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=reviewer-scorecard.yml)
- [scenario idea](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=scenario-idea.yml)
- [docs or template feedback](https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=docs-template-feedback.yml)

Use those only when you have a public-safe note. Do not ask people to paste private logs or transcripts.

## What Not To Share

Do not share:

- private logs
- customer or user details
- account, billing, wallet, session, or auth details
- private repo names
- screenshots with private context
- raw assistant transcripts
- claims that the kit solves security, compliance, or production review

If the real story matters, rewrite it as a fake one first.

## Slightly Longer Note

```text
I made a small public repo for coding-agent workflow guardrails.

It is not a security framework. It is a practical kit for the stuff that goes wrong when an assistant moves too fast:

- editing the wrong folder
- mixing private context into public docs
- treating urgency as approval
- publishing or submitting before a clear stop
- saying done without evidence

The most useful feedback would be one boring failure mode that should become a fake eval scenario.

Repo:
https://github.com/BobsCrispyDuck/ai-workflow-safety-kit

Useful feedback link:
https://github.com/BobsCrispyDuck/ai-workflow-safety-kit/issues/new?template=reviewer-scorecard.yml
```

## Cadence

Share once, then wait.

If nobody responds, improve the repo. Do not immediately repost the same thing somewhere else.

For asking one relevant person or small group, use `docs/review-request.md`.
