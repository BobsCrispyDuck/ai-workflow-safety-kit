# Repo Tour

If you just landed here, pick the amount of time you have.

## Three Minutes

Read:

- `docs/one-page-kit.md`
- `docs/known-limits.md`

You should leave knowing:

- what the kit is trying to stop
- what it does not claim to solve
- what kind of work should pause for approval

## Ten Minutes

Read:

- `docs/one-page-kit.md`
- `docs/try-it-now.md`
- `docs/reviewer-brief.md`
- `evals/coverage.md`

Try:

```text
one prompt from docs/reviewer-prompts.md
```

You should leave with one useful note:

```text
This catches X, but it misses Y.
```

## Thirty Minutes

Read:

- `templates/README.md`
- `checklists/README.md`
- `docs/example-result-receipts.md`
- `evals/rubric.md`
- `docs/issue-guide.md`

Try:

```text
one fake scenario from evals/scenarios.jsonl
```

You should leave knowing whether the stop points are clear enough to copy into your own workflow.

## If You Want To Copy It

Use:

- `docs/starter-pack.md`
- `templates/work-session-check.md`
- `templates/project-root-check.md`
- `templates/private-public-data-split.md`

Start small. Do not copy the whole repo unless you actually need the whole repo.

## If You Want To Help

Use:

- `docs/reviewer-scorecard.md`
- `docs/issue-guide.md`
- `docs/first-good-issues.md`
- `docs/review-request.md`

The best help is one boring failure mode, rewritten as a fake scenario.

## Do Not Bring

Do not bring:

- real logs
- secrets or keys
- customer or user details
- private repo names
- account, billing, wallet, session, or auth details
- screenshots with private context
- raw assistant transcripts

Rewrite real stories as fake setups before sharing them.
